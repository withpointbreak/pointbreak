Review stages and the CLI

Work -> Claims -> Evidence -> Questions -> Call maps to capture/revision/inspect, then observation, validation, input-request, and assessment. attention lists outstanding judgment; association records where the same revision landed. Validation is evidence, never a verdict or merge gate.

Review is local, read-only, and advisory: it shows and copies commands but never runs them. Copied commands keep visible placeholder tokens like <your-track> — replace each placeholder before running.
